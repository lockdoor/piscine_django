Ex01: Template engine.

Djangoâ€™s default template engine is designed to separate logic from presentation. It allows developers to insert dynamic content into HTML templates and control the rendering of that content using template tags and filters.

Key Features

1. Blocks

Blocks are used in templates to define sections that can be overridden in child templates when using template inheritance (i.e., extending a base template). A block is defined using {% block block_name %} ... {% endblock %}.

<body>
  {% block content %}
  <!-- Content will be injected here by child templates -->
  {% endblock %}
</body>

2. Loops (for ... in)

Django templates support the for loop to iterate over a list or any iterable object. This is useful for rendering lists of items, such as displaying a list of blog posts or products.

<ul>
  {% for item in items %}
    <li>{{ item.name }}</li>
  {% endfor %}
</ul>

3. If Control Structures

The if tag allows you to implement conditional logic in templates. You can check if a variable is true, false, empty, or compare it to another value.

{% if user.is_authenticated %}
  <p>Welcome, {{ user.username }}!</p>
{% else %}
  <p>Please log in.</p>
{% endif %}

4. Display of Context Variables

Context variables are passed from the view to the template and displayed using double curly braces {{ variable_name }}. These can represent any data, such as strings, numbers, lists, or objects.

<p>Hello, {{ name }}!</p>

Example Workflow

  1. View: The view passes context variables (e.g., {'name': 'Alice', 'items': items_list}) to the template.
  2. Template: The template uses blocks, loops, and conditionals to structure the HTML output and dynamically insert the passed context.
  3. Rendered Output: The final rendered HTML is sent to the user's browser, displaying the content based on the context variables and logic defined in the template.
