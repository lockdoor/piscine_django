Ex01: Display process of a static page.

In Django, the URLs, views, and templates work together to handle web requests and responses:

  1. URLs: Define the patterns for the different pages in your app. In urls.py, you map each URL to a specific view function using the path() function.
  2. Views: Views are Python functions (or classes) in views.py that handle the logic for what happens when a user visits a URL. They process requests and return responses, often in the form of rendered HTML.
  3. Templates: Templates are HTML files where you define the structure and layout of the web page. Django uses the template engine to dynamically insert content into these HTML files before sending them as a response to the user.

Example flow:

  • URL pattern in urls.py matches a request.
  • Corresponding view in views.py is executed.
  • The view processes data and renders a template with dynamic content.

This allows for a clean separation of logic, structure, and routing in a web application.
